* Maintain active sessions
* Store session memory reference
* Allocate sandbox
* Initialize AI context memory

→ Sandbox Orchestrator

### Responsibilities:

* Spawn Docker container
* Apply:

  * CPU limit
  * Memory limit
  * No outbound networking
* Auto-destroy on session end

```

```

### Sends To:

→ Session Manager

# [4] COMMAND INTERCEPTOR

### Responsibilities:

* Capture every terminal command
* Timestamp it
* Send command to AI Reasoning Core
* Never directly execute on host

### Data Sent:

```

  command,
  timestamp
}
```

# 1. PROJECT EXPLANATION

## 1.1 Problem Statement

Traditional cybersecurity systems are reactive. Attackers breach networks and remain undetected for long periods. Static honeypots collect limited intelligence and do not adapt to attacker behavior.

## 1.2 Proposed Solution

Build an **Autonomous AI Deception & Attribution Engine** that:

1. Engages attackers in real time via SSH.
2. Dynamically adapts the fake environment based on attacker behavior.
3. Uses AI to infer attacker intent and sophistication.
4. Maps behavior to MITRE ATT&CK techniques.
5. Generates autonomous threat intelligence reports.
6. Provides a live SOC dashboard.

The AI is not just generating terminal text.
It is acting as:

* A deception orchestrator
* A behavioral analyst
* An attribution engine
* An automated SOC assistant

---

# 2. SYSTEM PHILOSOPHY

This system must:

* Never expose the real host environment.
* Treat each attacker as an independent session.
* Use AI reasoning to adapt dynamically.
* Log everything in structured form.
* Generate intelligence, not just fake files.

# 5. MODULE RESPONSIBILITIES

---

# 5.1 gateway/

### ssh_server.py

* Accept inbound SSH connections.
* Fake authentication.
* Generate session_id.
* Forward session to Session Manager.

---

# 5.2 session/

### session_manager.py

* Maintain active sessions dictionary.
* Initialize AI memory context.
* Link container_id to session_id.
* Handle session cleanup.

Session Object:

```
Session:
    session_id
    source_ip
    container_id
    working_directory
    command_history
    ai_profile
    threat_score
```

---

# 5.3 sandbox/

### docker_manager.py

* Spawn isolated container per session.
* Apply CPU/memory limits.
* Disable outbound networking.
* Destroy container on termination.

---

# 5.4 interception/

### command_interceptor.py

* Capture every terminal command.
* Timestamp.
* Send to AI Core.
* Return AI-generated response to attacker.

---

# 5.5 ai_core/

This is the brain of the system.

---

## llm_client.py

* Handles communication with LLM API.
* Maintains per-session memory context.

---

## intent_inference.py

Input:

* Last N commands
* Command timing
* Classification tags

Output:

```
{
  attacker_type,
  primary_objective,
  sophistication_level,
  confidence
}
```

---

## environment_shaper.py

* Modifies visible filesystem.
* Injects relevant fake sensitive files.
* Keeps environment consistent.

---

## response_generator.py

* Generates realistic terminal outputs.
* Ensures stateful directory tracking.
* Streams responses.

---

## mitre_mapper.py

```
{
```

# 6. COMPLETE RUNTIME FLOW

1. Attacker connects via SSH.
2. Session created.
3. Docker sandbox spawned.
4. Command intercepted.
5. AI Reasoning Core processes:

   * Intent inference
   * Environment shaping
   * MITRE mapping
   * Threat scoring
   * Response generation
6. Response returned to attacker.
7. Telemetry logged.
8. Dashboard updated in real-time.
9. On session end:

   * AI report generated
   * Container destroyed
   * Session archived

# 7. CONCURRENCY MODEL

* Async SSH server
* One Docker container per session
* Independent AI memory per session
* Thread-safe session store
* Redis optional for caching

# 8. SECURITY CONSTRAINTS

* Containers run as non-root.
* No outbound internet from sandbox.
* API keys stored securely.
* Logs encrypted.
